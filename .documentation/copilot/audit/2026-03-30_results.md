### Security Checklist

- [ ] **CSRF protection on all `[HttpPost]` endpoints** — 10 violations
- [ ] **SQL identifiers use interpolation not concatenation** — 4 violations
- [x] No `Console.Write*` in production code
- [x] Sensitive paths sanitized via `DatabaseConfiguration`

### Detailed Security Issues

**SEC1–SEC10**: All 10 `[HttpPost]` endpoints in `HomeController` (lines 39, 188, 267, 291, 393, 420, 518, 661)
and `UnifiedDataController` (lines 139, 191) are missing `[ValidateAntiForgeryToken]`.

**Recommendation**: Add `[ValidateAntiForgeryToken]` to all 10 POST actions. For AJAX endpoints (`UploadApi`,
`GetUnifiedData`) that use `fetch()` or `XMLHttpRequest`, configure the Antiforgery middleware for header-based
validation (`X-CSRF-TOKEN`) or use `[IgnoreAntiforgeryToken]` only where the endpoint is intentionally
stateless and carries no authorization.

## Structured Logging Analysis

### Finding: LOG1 (MEDIUM)

**Principle VII** requires the web application to use **Serilog** as the logging provider via `UseSerilog()`.

- `Serilog.AspNetCore` (v10.0.0) is referenced in `sql2csv.web/Sql2Csv.Web.csproj` ✅
- `UseSerilog()` is **not called** in `sql2csv.web/Program.cs` ❌
- All Core services use correct `ILogger<T>` with structured message templates ✅

**Recommendation**: Add Serilog bootstrap configuration to `Program.cs`:

```csharp
builder.Host.UseSerilog((context, services, configuration) =>
    configuration
        .ReadFrom.Configuration(context.Configuration)
        .ReadFrom.Services(services)
        .WriteTo.Console());
```

## SQL Safety Analysis

All 4 violations are in `Sql2Csv.Core/Services/UnifiedAnalysisService.cs`:

| Line | Current (VIOLATION) | Required Fix |
